## 1. Concept & Vision

一个直观的可视化压测 Demo，让用户通过浏览器点击即可发起大量并发请求，实时观察 async 和 sync 端点在高并发下的性能差异。目标是让面试者通过亲身体验 + 原理图解，牢牢掌握 FastAPI（及通用异步框架）的高并发核心优势。

## 3. 核心端点设计

### Python (FastAPI, 端口 8091)

| Method | Path | 实现 | 模拟内容 |
|--------|------|------|----------|
| GET | `/async-llm` | `async def` + `await asyncio.sleep(2)` | 异步 IO 等待（不阻塞事件循环）|
| GET | `/sync-llm` | `def` + `time.sleep(2)` | 同步阻塞等待（独占 worker）|

### Node.js (Express, 端口 8090)

| Method | Path | 实现 | 模拟内容 |
|--------|------|------|----------|
| GET | `/async-llm` | `async handler` + `await sleep(2000)` (bluebird) | 异步 IO 等待 |
| GET | `/sync-llm` | sync handler + `while (Date.now() < start + 2000)` | 同步阻塞（卡死事件循环）|

## 5. 代码注释要点（面试解释）

### 5.1 当 10 个并发请求同时打进来

**async-llm (async def + await asyncio.sleep)**：
- 10 个请求同时到达事件循环
- `asyncio.sleep(2)` 让出控制权，事件循环立即调度下一个请求
- 2 秒后所有 10 个 sleep 同时完成
- **总耗时：~2 秒**

**sync-llm (def + time.sleep)**：
- 第 1 个请求获得 worker，sleep(2) 阻塞 2 秒
- 第 2-10 个请求排队等待（因为 uvicorn 默认单 worker）
- **总耗时：10 × 2 = 20 秒**

### 5.2 为什么 async def 里用 time.sleep 是灾难

```python
async def bad_example():
    time.sleep(2)  # ❌ 阻塞事件循环！
    # 这 2 秒内，整个服务器无法处理任何其他请求
    # 所有协程全部陪葬
```

**原理**：Python 的事件循环依赖协程在 await 点让出控制权。`time.sleep` 是 C 级别阻塞，不释放 GIL 也不让出事件循环，导致事件循环完全卡死。

**正确写法**：
```python
async def good_example():
    await asyncio.sleep(2)  # ✓ 挂起协程，事件循环可以调度其他任务
```

### 5.3 Node.js sync handler 的灾难

```javascript
app.get('/sync-llm', (req, res) => {
  const start = Date.now();
  while (Date.now() < start + 2000) {}  // ❌ 死循环阻塞事件循环
  res.send('done');
});
```

Node.js 单线程事件循环被这个 while 阻塞，所有其他请求（包括 async handler）全部卡住。

## 6. 压测结果预期

### Python FastAPI (50 并发)

| 端点 | 总耗时 | QPS |
|------|--------|-----|
| async-llm | ~2.1s | ~24 req/s |
| sync-llm | ~100s+ | ~0.5 req/s |

### Node.js Express (50 并发)

| 端点 | 总耗时 | QPS |
|------|--------|-----|
| async-llm | ~2.1s | ~24 req/s |
| sync-llm | ~100s+ | ~0.5 req/s |

**注意**：sync-llm 在 Node 中尤其灾难，因为 Express 默认单线程，sync handler 会连带阻塞 async handler。

## 7. 面试 3 要点（输出在 README 和页面）

1. **事件循环 + 协程调度**：async def 中的 `await` 让出控制权给事件循环，单线程即可高并发处理大量 IO 等待
2. **sync def = 独占线程**：普通函数会独占 worker 进程/线程，10 个并发 = 串行等待，无法复用
3. **严禁在 async 中用 blocking I/O**：`time.sleep`、`同步文件读取` 会阻塞事件循环，导致所有协程陪葬

## 8. Design Principles

- **零构建**：纯 CDN，无 webpack/vite
- **依赖隔离**：每个 demo 依赖在自己目录下
- **教学优先**：可视化是为了辅助理解原理，不是炫技
- **两语言对称**：Node.js 和 Python 行为一致，方便对比学习
- **直观冲击**：50 并发 + QPS 大数字，差距一目了然
