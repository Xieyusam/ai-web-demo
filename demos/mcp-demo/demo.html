MCP 统一教学演示

Model Context Protocol - Client-Server 架构对比

1 MCP 架构概述

Python Server Port 8100
  • • FastAPI 框架
  • • SSE (Server-Sent Events)
  • • JSON-RPC 2.0 协议
  • • Uvicorn ASGI 服务器
Node.js Server Port 8101
  • • Express.js 框架
  • • SSE (Server-Sent Events)
  • • JSON-RPC 2.0 协议
  • • Native HTTP 服务器

2 传输层对比: stdio vs SSE

stdio Transport

// Claude Desktop 插件场景
Claude Desktop ──stdin──► MCP Server
                              ◄──stdout──
  • 适用: Claude Desktop 插件、本地进程通信
  • 特点: 标准输入/输出管道，单向通信
  • 优点: 低延迟、无网络开销

SSE Transport

// 浏览器/Web 应用场景 (本演示)
Browser ──POST /mcp──► MCP Server
                              ◄──SSE /sse──
  • 适用: 浏览器应用、远程服务、Web界面
  • 特点: HTTP + SSE、双向通信
  • 优点: 支持远程调用、便于调试

3 MCP vs @tool 装饰器对比

特性 MCP Protocol @tool Decorator
工具定义 JSON Schema 格式 Python 函数包装器
工具发现 tools/list RPC 调用 直接函数调用
传输方式 stdio / SSE / HTTP 直接函数调用
跨语言支持 ✓ Yes (通用协议) ✗ Python only
典型场景 AI 与工具交互 Python 脚本内部
协议开销 JSON-RPC 2.0 格式 无 (直接调用)
