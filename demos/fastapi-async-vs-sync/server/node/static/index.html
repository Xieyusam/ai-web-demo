📊 什么是 QPS？

QPS = Queries Per Second

每秒查询率，衡量系统处理能力的核心指标

QPS = 总请求数 ÷ 总耗时（秒）
例如：10 个请求用了 2 秒 → QPS = 10 ÷ 2 = 5 req/s
为什么重要？
  • • QPS 越高 = 服务器并发处理能力越强
  • • 10 个用户同时访问：QPS 高 = 秒响应，QPS 低 = 排队等待
  • • 面试必问：高并发系统设计的第一步就是提升 QPS

🔬 技术原理详解

1. async handler + await

Promise 让出事件循环

app.get('/', async (req, res) => {
await sleep(2000);
});
10 并发： 全部 ~2s 完成 ✓
2. sync handler + while

阻塞事件循环

app.get('/', (req, res) => {
while (now < start + 2000) {}
});
10 并发： 10 × 2 = 20s ✗
3. sync 中调用 async ❌

Express 单线程被阻塞

// 同步代码阻塞
while (...) {}
// async 也无法执行
后果： 整个服务卡死

🎯 核心概念：Node.js 事件循环

同步（sync）：
请求1 ──▶ while循环 ──▶ 完成
请求2 ───────────▶ 等待中...
请求3 ───────────▶ 等待中...
单线程被阻塞，所有请求排队
异步（async）：
请求1 ──▶ await ─┼──▶ 完成
请求2 ──▶ await ─┼──▶ 完成
请求3 ──▶ await ─┼──▶ 完成
事件循环调度，同时完成

🎤 面试 3 大要点（必背）

1️⃣
事件循环 + 非阻塞 I/O

Node.js 单线程事件循环，async/await 让出控制权，单线程即可高并发处理大量 IO。10 个请求同时 await，全部 ~2s 完成。

2️⃣
sync 代码 = 事件循环杀手

同步 while 循环或计算密集型代码会阻塞事件循环，导致所有请求（包括 async）全部卡死。单线程无法并行。

3️⃣
CPU 密集型需要 Worker Threads

对于计算密集型任务（加密、压缩、AI推理），需要用 Worker Threads 或 child_process 放到独立线程执行。

SSE 方案：服务器内部并发调用，绕过浏览器连接数限制

async 全部同时开始，sync 依次执行
