🎯 面试 3 要点（必背）

1. 事件循环 + 协程调度

async def 中的 await 让出控制权给事件循环，单线程即可高并发处理大量 IO 等待。10 个并发请求同时打进来，所有 asyncio.sleep(2) 并行执行，总耗时 ~2 秒。

2. sync def = 独占线程

普通 def 函数会独占 worker 进程/线程。10 个并发请求 = 10 × 2 = 20 秒串行等待，无法复用。uvicorn/FastAPI 默认单 worker，严重退化。

3. 严禁在 async 中用 blocking I/O

time.sleep()、同步文件读取会阻塞事件循环，导致所有协程陪葬。生产环境必须用 await asyncio.sleep() 或 run_in_executor()。

💡 提示：建议先测试 async（绿色），感受"瞬间完成"的体验，再测试 sync（红色）体验串行阻塞

⚠️ sync-llm 测试时请耐心等待，100 并发可能需要约 3-4 分钟
